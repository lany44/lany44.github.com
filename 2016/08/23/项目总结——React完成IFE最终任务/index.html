项目总结——React完成IFE最终任务

Posted by lany44 on 2016-08-23

前言

3月份的时候拉着寝室长还有室友－隐形君X组了个团去参加百度ife前端学院，一边充电，一边努力完成任务。一路上由于大家水平相差太多，所以各自为战。结束时自己虽然只完成到第三阶段，但也坚定了前端这条道路，这里先由衷的感谢下百度EFE团队。
IFE结束了，但那最终任务却在脑海里一直挥之不去，终于自己在暑假废了些心思把她拿下了。/w\

任务要求
源码 在线demo

写了两周，参考第一名颜适的设计和部分实现。同样使用React将任务做成了SPA。还有很多可以优化的地方，但自己懒癌发作，打算就此搁置。

技术体系

  • JavaScript: React + Redux + React-Router
  • CSS: CSS-Modules + Sass + PostCSS
  • Graph: Recharts
  • Build: NPM + Babel + Webpack

js方面拥抱ES6，且采用了React技术栈，用Redux维护分发state，用React-Router做页内跳转。
css使用Sass＋PostCSS做buff，用CSS-Modules解决模块化。
可视化参考第一名简单使用了Recharts。

不了解React的同学还是先去读文档吧

State

这里介绍下Redux维护的State
大体上用editing中间态做缓冲，提交时推入list并存于localStorage。

  • questionnaires包含了问卷系统的主要信息。
    • list是记录每张问卷信息的数组，存于localStorage。
      • title为问卷标题。
      • deadline为问卷截止时间。
      • status为问卷当前状态。
      • questions是记录每个问题信息的数组。
        • type为问题类型。
        • content为选择题的标题，或者文本题的问题内容。
        • 若为选择题，则有options记录每个选项的内容。
        • 若为文本题，则有isRequired记录该题是否必填。
      • data为记录当前问卷所有数据的二维数组，第一维表示问卷填写数据的序号，第二维表示问题的序号。
    • editing记录了编辑时的中间状态。
      • id为当前编辑问卷的序号。
      • question标记title，可编辑时title作为－1个问题。
      • title为当前编辑问卷的标题。
      • deadline为当前编辑问卷的截止时间。
      • questions是记录当前编辑问卷的问题信息的数组，结构同上。
      • data记录填写中的问卷的数据，结构同上。
  • dialog, calendar略。

Components

简单介绍下自己写的组件。

  • Table，Column，SortTableTh — 表格三剑客

    安 利 下面这篇文章 ↓ ↓ ↓

    详细请看React 实现 Table 的思考，相当有深度 。【强～/w\

  • Dialog

    Dialog逻辑并不复杂，props接收一个dialog对象包括弹窗的必要信息{ title, info, onConfirm, onCancel }其中2个回调函数为可选参数。由于dialog内容是可变的，所以交给了reducer维护，但每次dialog发生变化时候会更新页内所有Dialog的props。虽然只会有必要那只会显示。。。非常的蠢TAT。

  • EditBox

    EditBox接收保存输入的Action－saveText，并且依靠question，option两个props来标记自己位置。

    每个EditBox各自维护着自己当前的显示状态，输入还是展示。

    主要是复用的很舒服，所以贴了出来。【组件化滴好啊/w\

  • Calendar

    日历的是耗时最久的地方，最终还是参考了手边微博送的RN的书上的日历逻辑，比较巧妙。

    摘了段代码，有兴趣的跟着走一遍吧。

其实还有地方可以抽象成Stateless Components，但我偷懒直接复制粘贴了。/w\
